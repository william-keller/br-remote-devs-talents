# BR Remote Devs Talents

A curated collection of professional resumes from talented Brazilian remote developers. This repository serves as a comprehensive talent pool for recruiters and organizations seeking skilled professionals across various technology stacks and experience levels.

## Overview

This repository contains 41 professionally curated resumes from Brazilian developers specializing in remote work. The collection spans multiple technologies including .NET, Angular, Cloud Platform Engineering, DevOps, Flutter, Java, Node.js, Oracle, Python, QA, React, React Native, SRE, Vue, and iOS development. Developers are organized by technology and experience level, making it easy for recruiters to find candidates matching specific requirements.

## How to Use This Repository

### For Recruiters

This repository is designed to facilitate talent acquisition for Brazilian remote developers. Recruiters can browse the directory by technology specialization and experience level to identify candidates matching their requirements. Each resume is directly accessible through GitHub links, enabling quick evaluation of qualifications.

### For Developers

If you are a Brazilian developer interested in contributing your resume to this talent pool, please follow the contribution guidelines below. Your profile will be visible to potential employers and recruiters seeking remote talent.

### Interactive Web Directory

An interactive web-based directory is available at the [BR Remote Devs Talent Directory](https://brtalents-44pj8gsw.manus.space) with advanced search and filtering capabilities. The web application supports both Portuguese and English interfaces, allowing users to search by developer name, filter by technology, and refine results by experience level.

## Contributing

To add your resume to this talent pool, follow these steps:

1. **Fork the repository**: Create a personal fork of this repository to your GitHub account.

2. **Create a directory structure**: Navigate to or create the appropriate technology folder. If it doesn't exist, create a new folder with your technology name. Within that folder, create a subdirectory for your experience level (e.g., `Sr`, `mid`, `Jr`).

3. **Add your resume**: Place your resume PDF file in the appropriate technology and level directory. Use a clear filename such as `FirstName_LastName.pdf` or `FirstName-LastName.pdf`.

4. **Commit and push**: Add your changes with a descriptive commit message such as "Add [Your Name] resume for [Technology] - [Level]".

5. **Submit a pull request**: Create a pull request to the main repository with a clear description of your addition. Include your name, technology focus, and experience level in the PR description.

6. **Review and merge**: The repository maintainers will review your submission and merge it if it meets the quality standards.

### Resume Guidelines

When submitting your resume, please ensure it meets the following criteria:

- **Format**: PDF files only (maximum 5 MB)
- **Content**: Include contact information, professional summary, technical skills, work experience, education, and certifications
- **Language**: Resumes may be in Portuguese or English
- **Accuracy**: Ensure all information is current and accurate
- **Professionalism**: Use a clean, professional layout with consistent formatting

## Repository Maintenance
